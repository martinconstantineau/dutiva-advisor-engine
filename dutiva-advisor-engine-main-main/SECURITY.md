# Security Policy

## Reporting a Vulnerability

Please report security issues privately via GitHub Security Advisories for this repository.

Include:

1. A clear description of the issue and impact.
2. Steps to reproduce.
3. Affected versions and environment details.
4. Any suggested remediation.

Do not open public issues for vulnerabilities that could expose users or systems.

## Scope

This policy covers:

- API request handling and response rendering gates.
- Dependency and supply-chain security.
- Secret exposure in source, logs, and runtime errors.
- CI/CD configuration and workflow security.
